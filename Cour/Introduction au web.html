Introduction au web

Le «World Wide Web», plus communément appelé «Web» a été développé au CERN (Conseil Européen pour la Recherche Nucléaire) par le Britannique Sir Timothy John Berners-Lee et le Belge Robert Cailliau au début des années 90.

À cette époque les principaux centres de recherche mondiaux étaient déjà connectés les uns aux autres, mais pour faciliter les échanges d'information Tim Berners-Lee mit au point le système hypertexte. Le système hypertexte permet, à partir d'un document, de consulter d'autres documents en cliquant sur des mots clés. Ces mots «cliquables» sont appelés hyperliens et sont souvent soulignés et en bleu. Ces hyperliens sont plutôt connus aujourd'hui sous le simple terme de «liens».

Cette première page web est toujours consultable à l'adresse suivante : http://info.cern.ch/hypertext/WWW/TheProject.html

Tim Berners-Lee développa le premier navigateur web (logiciel permettant de lire des pages contenant des hypertextes), il l'appela simplement «WorldWideWeb». Il fallut attendre 1993 et l'arrivée du navigateur web «NCSA Mosaic» pour que le web commence à devenir populaire en dehors du petit monde de la recherche.

Une chose très importante à bien avoir à l'esprit : beaucoup de personnes confondent «web» et «internet». Même si le «web» utilise internet, les deux choses n'ont rien à voir puisqu'«internet» est un «réseau de réseaux» s'appuyant sur le protocole IP alors que le web est une application d’internet qui permet le partage de documents liés entre eux et appelés «pages web» : une page web peut contenir du texte, des images, des programmes, des liens vers d’autres pages web... Le web fonctionne en mode client/serveur au-dessus de l’architecture TCP/IP : l'application est répartie entre le client et le serveur qui dialoguent selon un protocole applicatif spécifique.

Le web repose sur 3 concepts :

  • L’URL (Uniform Ressource Locator) qui permet au client de désigner la page demandée sous la forme d'une adresse répondant aux trois questions suivantes.
    Comment ?Où ?Quoi ?
    http://etu.univ-lyon1.fr/licence/lifasr2.html
  • HTTP (HyperText Transfer Protocol) : permet de définir le format et la signification des messages échangés entre le client et le serveur (protocole). Ces messages sont de deux types :
    • requête HTTP, du client vers le serveur, pour demander une page web ;
    • réponse HTTP : du serveur vers le client, pour répondre au navigateur.
  • HTML, CSS, PHP, Javascript... : les langages du web.
    • le HTML (HyperText Mark up Language) permet de décrire le contenu d’une page web, interprété par le navigateur web pour afficher la page et demander les objets incorporés ;
    • le CSS (Cascading Style Sheets) permet de définir les styles de la page (format, couleurs, positions...) ;
    • le PHP (initialement Personal Home Page, mais aujourd'hui PHP: Hypertext Preprocessor) permet l’exécution d'un programme par le serveur ;
    • le Javascript permet l’exécution d'un programme par le navigateur.

Par ailleurs on trouve autre chose que le web sur internet, par exemple, les emails (méls en français) avec le protocole SMTP (Simple Mail Transfer Protocol) et les transferts de fichiers avec le protocole FTP (File Transfer Protocol).
